# 영상공방 — 자료 강화(RAG·딥리서치) 가이드 (사용자 교육용)

> 대본 생성 **전에** 누르는 "준비 도구". 도커·API 키·외부 서비스 **0개**, 전부 로컬에서 동작.
> 화면에는 한 줄로만 안내하고, 자세한 이유는 이 문서로 교육한다.

---

## 왜 이 도구들을 두었나 (odysseus 툴을 쓰는 이유)

영상공방은 **odysseus** 위에 만들었다. odysseus에는 이미 검증된 로컬 엔진이 들어 있다:
- **로컬 임베딩**(`src/embeddings.py`의 FastEmbed, ONNX) — 서비스/도커 없이 문장을 벡터로 변환
- **딥리서치 방법론**(`src/deep_research.py`) — "분해 → 검색 → 종합"의 다단계 조사 패턴
- 메모리/검색/문서색인 등

이 자산을 **새로 만들지 않고 재사용**하는 것이 영상공방이 odysseus 위에 설 이유다.
다만 **웹 검색(SearXNG)과 ChromaDB는 도커가 필요**하므로 쓰지 않는다. 대신 같은 엔진의
**로컬 부분만** 떼어 가볍게 얹었다(`services/vodstudio/local_rag.py`).

## 버튼 1 — 📚 RAG (자료 학습)

**무엇:** 첨부한 법령·문서를 그 자리에서 조문 단위로 잘라 임베딩하고 색인한다.
(`local_rag.build_index` → FastEmbed 임베딩 + numpy 코사인 검색, 디스크 저장)

**화면 한 줄:** "첨부 자료를 학습해 슬라이드마다 관련 조문만 근거로 씁니다."

**왜 쓰면 달라지나:**
1. **큰 문서가 안 잘린다.** 예전엔 법률+시행령+시행규칙 전문을 통째로 프롬프트에 넣어
   Windows 명령줄 한계(약 32KB)를 넘겨 `WinError 206`으로 실패했다. RAG는 슬라이드 주제별로
   **관련 조문만** 뽑아 넣으므로 자료가 아무리 커도 안전하다.
2. **정확도↑.** "사업주의 안전조치 의무"를 만들 때 실제 **제38조(안전조치)** 조문이 근거로
   붙는다(검증된 동작). 모델이 기억에만 의존해 지어내는 일이 줄어든다.
3. **자료가 많을수록 유리.** 여러 법령·고시·지침을 함께 넣어도 슬라이드마다 알맞은 근거를 찾는다.

## 버튼 2 — 🔬 딥리서치 (자료 심층분석)

**무엇:** 웹이 아니라 **방금 학습한 자료**를 LLM(codex/agy)이 다단계로 분석해
"핵심 쟁점 · 구조 · 근거" 리서치 브리프를 만든다. (odysseus deep_research의 분해→종합
방법론을 로컬 RAG에 적용)

**화면 한 줄:** "자료의 핵심 쟁점을 먼저 정리해 60장의 뼈대를 잡습니다."

**왜 쓰면 달라지나:**
1. **단순 요약이 아니다.** 주제를 쟁점으로 쪼개고(예: 적용범위/사업주 의무/벌칙/안전조치…),
   각 쟁점에 RAG로 근거를 모아 구조화한다 → 60장이 **논리적 흐름**을 갖는다.
2. **누락 방지.** 자료에 있는데 빠뜨리기 쉬운 항목을 브리프 단계에서 먼저 끌어올린다.
3. 브리프가 대본 생성의 **설계도**가 되어, 길어도 일관성이 유지된다.

> 참고: "웹 딥리서치"(인터넷 검색 기반)는 SearXNG = 도커가 필요해 현재 비활성이다.
> 검색 백엔드를 붙이면 켤 수 있도록 코드에 자리만 남겨둔다.

## 권장 사용 순서

1. ① 대본 탭에서 법령 파일들을 첨부
2. **📚 RAG** 눌러 학습(색인)
3. (선택) **🔬 딥리서치** 눌러 쟁점 브리프 생성
4. 슬라이드 수(예: 60) 정하고 **✦ 대본 생성** — RAG 근거 + 브리프로 생성(206 없음)
5. 이후 ② 이미지 → ③ 음성/자막 → ④ 영상
