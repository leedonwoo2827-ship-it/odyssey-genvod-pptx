# 영상공방 — 대본 검수·수정 워크플로 (사용자 교육용)

> ① 대본을 만든 뒤, **근거 자료(RAG)와 대조해 사실관계를 다듬는** 단계.
> 버튼 2개: **✅ 대본 자동 검수 / 🟡 검수 반영 수정**.
> 둘 다 RAG(📚 자료 학습)로 근거를 먼저 색인해야 동작한다. (딥리서치는 불필요)

---

## ⭐ 핵심 한 줄 — 이것만 기억하세요

> **🔴(부정확)만 "해당 없음"이면 끝 — 바로 번들 저장(대본 확정).**
> **🟡(과장/모호)는 0이 안 됩니다. 정상입니다. 0을 추구하지 마세요.**

설명 문장은 거의 다 "해석/요약"이라 검수기가 **돌릴수록 새로운 뉘앙스를 계속 잡아냅니다(점근선처럼 0에 안 닿음)**. 거슬리는 몇 개만 직접 고치고, 나머지 🟡는 두셔도 됩니다.

> 특히 이 도구는 **일반 교육영상**용입니다. 법률·의료처럼 한 글자도 틀리면 안 되는 산출물이 아니라, 사람이 최종 검수·확정하는 보조 대본이므로 🟡에 엄격할 필요가 없습니다.

---

## 한눈에 보는 흐름

```
📚 자료 학습(RAG)  →  ✦ 대본 생성  →  ✅ 검수  →  (필요시) 🟡 수정 / 직접 수정  →  번들 저장(확정)
```

- **✅ 대본 자동 검수**: 대본을 RAG 근거(자료 발췌)와 대조해 🔴/🟡/🟢로 분류.
- **🟡 검수 반영 수정**: 검수 결과(과장/모호 완화 위주)를 대본에 한 번 자동 반영. *(검수 후에만 버튼이 나타남)*
- **직접 수정**: 거슬리는 문구만 textarea에서 손으로 고치는 게 가장 확실. 다시 ✅ 검수를 누르면 **현재 대본을 그대로 다시 읽어** 대조하므로, 고친 항목은 다음 검수에서 사라진다.

> ※ 예전의 **"✨ 자동 정리(검수+수정 반복)"** 버튼은 제거했다. 🟡을 끝까지 0으로 만들지 못해(애초에 0이 안 됨) "자동인데 자동이 아닌" 혼란만 줬기 때문. 이제는 **검수 → 필요한 것만 수정**의 단순 흐름이다.

## 검수 결과 3색의 의미

- 🔴 **부정확/근거없음** — 자료와 명백히 어긋나는 서술/수치. **반드시 고친다.** (이게 0이면 확정 OK)
- 🟡 **과장/모호** — 자료가 입증한 것보다 넓게 말함(해석을 단정처럼). **완화 권장이지 필수 아님.** 0 안 됨.
- 🟢 **누락** — 자료엔 있는데 대본에 빠진 내용. 추가는 **선택**(신뢰도 보강용, 분량 채우기용 아님).

### 🔴 오탐 주의 — 가장 중요한 함정
검수는 **학습된 발췌 안에서만** 대조한다. **자료를 일부만 RAG에 넣으면**, 실제로는 자료에 있는 수치도 "근거 없음 🔴"으로 잡히는 **오탐**이 대량 발생한다.

→ **해결: 원본 전체를 📚 자료 학습에 넣고 검수.** (`enrich.review_queries`가 대본 전체 구간을 질의로 만들어 자료 전체를 훑는다 — `routes/vodstudio_routes.py`의 `review-script`, `services/vodstudio/enrich.py`.)

> ⚠️ **불러오기(번들 load) 후에는 RAG가 초기화된다.** 검수하려면 **자료 파일을 다시 첨부하고 📚 자료 학습을 한 번 더** 눌러야 한다(RAG 색인은 번들에 저장되지 않음).

## "기조" 4원칙 — 고칠지 말지 객관 기준 (특히 교육물)

수정 여부는 **취향이 아니라 "과장인지"**로 판단한다. 다음을 어기면 고치고, 다 지키면 멈춘다.

1. 자료가 입증한 것만 단정한다 (해석을 "보고서가 말한다"처럼 쓰지 않기)
2. **상관 ≠ 인과** (예: "전문성↔성공 상관"을 "전문성을 키우면 성공한다"로 쓰지 않기)
3. "보고서가 말한다" vs "이렇게 해석된다" 구분
4. 비중·규모를 정확히 (예: 33%는 과반이 아니므로 "중심"이라 쓰지 않기)

→ 검수가 위 4개와 무관한 "더 예쁘게 쓸 수도" 수준만 지적하면 = 취향 → **거기서 끝.**

## 실제 예시 — 새뮤얼슨 경제학 5부 19장 (60장 대본)

검수 결과(아래는 실제 화면):

```
🔴 부정확/근거없음: 해당 없음

🟡 과장/모호:
- 슬라이드 1·60 — 거시경제학 핵심을 "경제 전체의 낭비를 줄이는 학문"으로 단정해 범위가 좁아짐
- 슬라이드 11 — 잠재 GDP를 "지속 가능한 최대 수준"으로만 설명하면 물리적 최대 산출량으로 오해될 수 있음
- 슬라이드 17 — "구직 활동이 없으면 통계상 제외"가 취업자까지 포함하는 것처럼 읽힐 수 있음
- 슬라이드 20 — 1982년 실제 산출량 약 4,000억 달러 부족 / 가구당 5,000달러 감소 수치는 발췌에서 미확인
- 슬라이드 26 — "보통 연간 1~3% 범위"는 발췌에서 직접 확인되지 않음
- 슬라이드 27 — 짐바브웨 2008년 약 연 2억%, 닭고기 1만→200억 짐바브웨달러는 발췌에서 미확인 세부
- 슬라이드 35 — "GDP 크기 자체를 좌우"는 정부지출 영향력을 다소 강하게 표현
```

**해석**:
- 🔴 **"해당 없음"** → 사실관계상 틀린 곳 없음 → **이미 확정해도 되는 상태.**
- 남은 🟡 7건은 전부 "해석을 좀 더 좁히거나 톤을 낮추라"는 권고. 안 고쳐도 무방.
- 거슬리는 것만 **직접 수정** 가능. 예: 슬라이드 35
  - `GDP 크기 자체를 좌우` → `GDP와 전체 지출 수준에 영향을 미친다`
  - 이렇게 바꾸고 **✅ 검수를 다시 누르면 → 슬라이드 35 항목만 사라진다** (현재 대본을 다시 읽으므로).
  - **단**: 35만 고치면 35만 빠지고 1·11·17·20·26·27은 그대로 남는다. 각각 고쳐야 그 줄이 사라진다.
  - **그리고** 전체 🟡이 0이 되는 일은 거의 없다 — 고치면 가끔 검수기가 다른 미묘한 표현을 새로 하나 잡기도 한다.

**교훈 2가지**:
1. 🔴가 잔뜩 떠도 먼저 **"자료를 전부 학습했는지"** 의심하라(오탐일 때가 많다).
2. 🟡은 끝까지 따라가지 마라. **🔴=0이면 확정**, 🟡은 신경 쓰이는 것만 손보면 된다.

## 모델 선택 팁 (① 상단 드롭다운 — ChatGPT Codex OAuth)

모델 설정은 **전역**이다(고른 순간부터 모든 단계에 적용). 단계 사이에 바꾸면 그다음 작업부터 반영된다.

| 단계 | 권장 | 이유 |
|---|---|---|
| 대본 생성 | **5.5** (품질 우선) | 콘텐츠 본체 |
| 검수 · 수정 | **5.4 (mini)** 로 통일 | 빠름. 검수·수정을 **같은 모델로 맞춰야** 재검수 시 🟡이 잘 줄어든다(모델 섞으면 기준이 달라 수렴 안 됨) |

- 단순함을 원하면 **처음부터 끝까지 5.4**도 충분하다(일반 영상이라 OK). 토글을 깜빡할 일이 없다.
- **주의**: 🟡 수정은 텍스트를 다시 쓰므로, 5.5 검수 + 5.4 수정처럼 섞으면 수정이 검수 기준을 다 못 맞춰 🟡이 안 줄어든다. **검수·수정 모델을 일치**시키는 게 핵심.
- 임베딩 모델은 1회만 로드되도록 캐싱됨 — 검수 속도 정상화.
