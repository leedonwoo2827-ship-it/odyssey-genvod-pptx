# 영상공방 — 대본 검수·수정·자동 정리 워크플로 (사용자 교육용)

> ① 대본을 만든 뒤, **근거 자료와 대조해 사실관계를 다듬는** 단계.
> 버튼 3개: **✅ 대본 자동 검수 / 🟡 검수 반영 수정 / ✨ 자동 정리(검수+수정 반복)**.
> 모두 RAG(📚 자료 학습)로 근거를 먼저 색인해야 동작한다.

---

## 한눈에 보는 흐름

```
📚 자료 학습(RAG)  →  ✦ 대본 생성  →  ✅ 검수  →  🟡 수정 / ✨ 자동 정리  →  📋 복사 → NotebookLM
```

- **✅ 대본 자동 검수**: 대본을 RAG 근거(자료 발췌)와 대조해 문제를 분류한다.
- **🟡 검수 반영 수정**: 검수 결과를 대본에 한 번 자동 반영(과장/모호 완화 위주).
- **✨ 자동 정리**: 검수→수정을 **최대 2회** 반복해 한 번에 다듬는다(아래 "왜 2회만" 참고).

---

## 검수 결과 3색의 의미

- 🔴 **부정확/근거없음** — 자료와 명백히 어긋나는 서술/수치. (반드시 고친다)
- 🟡 **과장/모호** — 자료가 입증한 것보다 넓게 말함(해석을 단정처럼). (완화 권장)
- 🟢 **누락** — 자료엔 있는데 대본에 빠진 내용. (추가는 **선택** — 신뢰도 보강용, 분량 채우기용 아님)

### 🔴 오탐 주의 — 가장 중요한 함정
검수는 **학습된 발췌 안에서만** 대조한다. 그래서 **자료를 일부만 RAG에 넣으면**,
실제로는 자료에 있는 수치도 "근거 없음 🔴"으로 잡히는 **오탐**이 대량 발생한다.

→ **해결: 원본 전체를 📚 자료 학습에 넣고 검수**한다. (관련 코드: `enrich.review_queries`가
대본 전체 구간을 질의로 만들어 자료 전체를 훑게 수정됨 — `routes/vodstudio_routes.py`의
`review-script`, `services/vodstudio/enrich.py`.)

---

## "기조" 4원칙 — 고칠지 말지 객관 기준 (특히 교육물)

수정 여부는 **취향이 아니라 "과장인지"**로 판단한다. 다음을 어기면 고치고, 다 지키면 멈춘다.

1. 자료가 입증한 것만 단정한다 (해석을 "보고서가 말한다"처럼 쓰지 않기)
2. **상관 ≠ 인과** (예: "전문성↔성공 상관" 을 "전문성을 키우면 성공한다"로 쓰지 않기)
3. "보고서가 말한다" vs "이렇게 해석된다" 구분
4. 비중·규모를 정확히 (예: 33%는 과반이 아니므로 "중심"이라 쓰지 않기)

→ 검수가 위 4개와 무관한 "더 예쁘게 쓸 수도" 수준만 지적하면 = 취향 → **거기서 끝.**

## ✨ 자동 정리는 어떻게 도나 (한 번 누르면 다듬고, 부족하면 또 누르기)

- **한 번 누르면** 검수→수정을 짧게 몇 번 다듬고, **깨끗해지면 멈춘다.** 끝까지 강제로
  돌리지 않는다. 남았으면 **다시 눌러 계속**하면 된다 — *없어질 때까지, 또는 간단한 것 하나만
  남을 때까지* 사용자가 페이스를 잡는다.
- 진행 상황을 **"처음 🔴N·🟡M건 있었는데 지금 없음"**으로 보여줘, *정리됐음을 눈으로 확인*하게 한다.
  ("있었는데 사라진 것"을 보는 게 안정감을 준다.)
- **왜 한 번에 길게 안 도나:** 매 수정마다 문장을 다시 쓰므로, 연속 재작성이 많으면 멀쩡한 곳에
  오타("피드백"→"피드작" 같은)나 미세 드리프트가 생긴다. 그래서 **한 번에 도는 횟수는 짧게** 잡고,
  더 필요하면 사용자가 직접 다시 눌러 통제한다.
- **🟡은 0까지 안 간다.** 설명 문장은 거의 다 "해석"으로 잡혀, 돌릴수록 검수기가 새 뉘앙스를
  계속 찾아낸다(점근선). 🔴가 없고 🟡이 취향 수준만 남으면 거기서 멈추면 된다.

> 슬라이드가 아주 많으면(예: 60장) 한 번에 도는 횟수·청크 동작을 더 조정할 수 있다 — 별도 튜닝.

## 실제 예시 — Anthropic "Agentic coding" 보고서 (60장 대본)

1. **처음 검수**: 🔴 다수("프롬프트당 10개 행동", "성공률 15/28%" 등이 근거에 없다고 표시).
   → 실제로는 **PDF에 다 있는 정확한 수치**였고, RAG에 **일부만** 학습돼서 생긴 **오탐**이었다.
2. **원본 전체를 학습 후 재검수**: 🔴 **"발견된 문제 없음"** — 대본이 사실관계상 정확함이 확인됨.
3. 남은 건 🟡 몇 건(전부 "과장/해석" 유형):
   - "업무 실행 환경" → "코딩 보조를 넘어 운영·기획·분석 등으로 확장" (범위 완화)
   - "고장 수정 중심"(33%) → "고장 수정 비중이 높았고 이후 감소" (과반 아님)
   - "전문성은 훈련 대상" → "전문성 신호가 성공과 연결되므로 교육 대상으로" (상관≠인과)
4. **✨ 자동 정리(또는 🟡 수정)** → 🟡도 사라짐. 🟢 누락(방법론·핵심 결론 등)은 선택 보강.

**교훈**: 🔴가 잔뜩 떠도 먼저 **"자료를 전부 학습했는지"** 의심하라. 대본이 틀린 게 아니라
검수 시야가 좁았을 때가 많다.

## 모델 선택 팁 (① 상단 드롭다운, 단계별 적용)

- **대본 생성**: 품질 중요 → `Gemini 3.x Pro` 또는 `Claude Sonnet 4.6` (느려도 1회)
- **검수 · 수정 · 자동 정리**: 빠름 우선 → `Gemini 3.5 Flash (High)` 권장
- 드롭다운은 **작업 시작 시점의 값**이 적용되므로, 단계 사이에 바꾸면 그다음 작업부터 반영된다.
- 임베딩 모델은 1회만 로드되도록 캐싱됨(`src/embeddings.py`) — 검수 속도 정상화.
