# 비주얼 스타일 13종 (NotebookLM 비주얼원고 · 슬라이드 디자인)

② **슬라이드 스타일**에서 스타일 하나를 고르면, 그 스타일이 다음 두 곳에 **공통 적용**됩니다.

1. **NotebookLM 렌더 코드의 디자인 시스템** — 슬라이드덱 전체 룩(Custom Visual Style 텍스트).
2. **슬라이드별 비주얼원고 프롬프트** — 흰 배경 삽화/메타포/인포그래픽(캡처해 PPTX에 삽입).

스타일 정의는 `data/visual_styles.json` 한 곳에서 관리합니다(추가/수정 시 UI·렌더코드·프롬프트에 동시 반영).

## 강도 3단계
| 강도 | 의미 |
|---|---|
| 은은하게 | 색·조명만 해당 스타일, 형태는 사실적 |
| 적당히 (기본) | 스타일 분명하되 정보 전달 우선 |
| 풀 스타일 | 해당 기법을 전면 적용 |

## 원칙 (저작권)
기법·질감·조명·구도만 차용합니다. 특정 작품의 **캐릭터·로고·고유 디자인은 재현하지 않습니다.**
비주얼원고 이미지에는 **글자(텍스트)를 넣지 않습니다** — 텍스트는 슬라이드(PPTX)가 담당합니다.

## 스타일 추가/수정
`data/visual_styles.json`의 `styles` 배열에 항목을 추가하면 됩니다.
- 필수: `id`, `name`, `def`, `keywords[]`
- `notebooklm: true` 여야 UI에 노출됩니다(상한 운영). `nlm_style`에 NotebookLM 네이티브명 또는 `Custom (...)` 매핑.
- `recommended: true` → ★ 표시, `default: true` → 첫 기본 선택.
