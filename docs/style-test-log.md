# 슬라이드 스타일 테스트 로그

NotebookLM 비주얼원고(흰 배경) 생성에 쓸 스타일을 **하나씩 실제로 테스트**해 확정한다.
데이터: [`data/visual_styles.json`](../data/visual_styles.json) 의 `notebooklm` 플래그로 UI 노출을 제어.

## 판정 기준
1. **흰 배경(#FFFFFF)** 이 잘 유지되는가 (캡처해 슬라이드에 얹기 좋아야 함)
2. NotebookLM 채팅/스튜디오에서 **렌더 코드가 거절 없이 실행**되는가
3. 생성된 비주얼이 **원고로 확정할 만큼 명확**한가 (너무 추상적이면 부적합)

## ✅ 테스트 중(활성) — 15종 전체 중 12종
플랫 벡터 · 인포그래픽/픽토그램 · 미니멀 라인아트 · 수채화 · 페이퍼 컷아웃 · 두들/스케치노트 · 시네마틱 에디토리얼 · 리소그래프 · 목판화/우키요에 · 클레이 렌더(소프트 3D) · 스톱모션/클레이메이션 · 실사 시네마틱 포토

> 흰 배경 유지가 좋은 편: **플랫 벡터 · 인포그래픽 · (미니멀) 라인아트** (2D 평면). 3D·수채·질감 계열은 배경에 그림자·미색이 남을 수 있어 캡처 후 배경 치환을 권장.

## ❌ 확정 제외 (사유)
| 스타일 | 사유 |
|---|---|
| **빈티지 에디토리얼(미드센추리 포스터)** | 생성된 비주얼이 **너무 추상적**이라 원고로 확정하기 어려움 |
| **아이소메트릭 3D** | NotebookLM **채팅창이 렌더 코드를 거절**(가드레일) |
| **스타일라이즈드 3D 애니메이션** | **#FFFFFF 강제가 안 됨** — 전역조명·그림자로 배경에 그라데이션이 남음 |

## 참고
- 렌더 코드가 채팅에서 거절되면: 재시도 2~3회, 또는 **스튜디오 "슬라이드 자료/동영상 개요 → 맞춤설정"** 에 붙여넣기.
- 스타일을 켜고 끄려면 `data/visual_styles.json` 해당 스타일의 `"notebooklm"` 값을 `true`/`false` 로 바꾼다(서버 재시작 필요).
